C#

¿Qué es C#?

C# es una evolución que hizo Microsoft a partir de los lenguajes C y C++, además también ha tomado cosas de otros lenguajes como java. Está orientado a objetos y con el tiempo adaptó las facilidades de creación de código que tenia otro de sus lenguajes más populares, Visual Basic, haciéndolo tan polivalente y fácil de aprender como éste, sin perder nada de la poténcia original de C. El lenguaje C# estaba hecho con la intención de uso en la plataforma .NET. En la versión .NET Core se ha reconstruido por completo su compilador, haciendo las aplicaciones un 600% más rápidas.

¿Cuáles son las características de C#?

C# no ha perdido la potencia original que tiene C, en otras palabras, se puede acceder a bajo nivel al núcleo de los sistemas operativos, se puede trabajar con punteros de memoria e interactuar con elementos físicos de los dispositivos, tarjetas gráficas o puertos USB. Además C# soporta archivos XML.

  • Sencillez: C# elimina los elementos que sean innecesarios en .NET. Por ejemplo no se incluyen los elementos poco útiles como los macros, herencias múltiples o la necesidad de un operador distinto del punto.
  • Modernidad: Incorpora de forma automática e intuitiva en su lenguaje elementos que se han demostrado con el paso de los años que han sido muy útiles para el desarrollo de aplicaciones.
  • Seguridad: Incorpora mecanismo para asegurar que los accesos a tipos de datos se lleven a cabo de forma correcta, por lo que se evita que generen errores difíciles de detectar.
  • Sistemas de tipos unificados: Los datos se almazenan en una base para que se pueden utilizar posteriormente.
  • Extensibilidad: posibilidad de agregar tipos de datos básicos, operadores y modificadores.
  • Compatibilidad: al mantener una sintaxis muy parecida a C y C++, java y más lenguajes facilita el movimiento del código.
  • Eficiencia: C# tiene muchas restricciones en el código pero se pueden saltar utilizando punteros.

¿Cómo utilizar C#?

Microsoft pone al alcance de la comunidad plataformas de desarrollo como Visual Studio Code (licencia gratuita) el cual es multiplataforma (para Windows, Linux y Mac OS) bajo el entorno .NET Core y también Visual Studio (de pago) bajo el entorno .NET Framework. La diferencia entre estos dos entornos es el destinatario final de las aplicaciones, siendo el último solo para Windows.
C# está diseñado principalmente para su uso en Virtual Studio ya que sido desarrollado especialmente para ello y soporta la carga y trabajo con su mismo lenguaje.

¿Cuál es su historia?

Andrés Hejlsberg decidió formar un equipo de trabajo en 1999 para crear un nuevo lenguaje de programación, Cool (C Object Oriented Language). Microsoft quiso mantener el nombre como definitivo pero por razones de registro de marca tuvo que echarse para atrás.
En julio de 2000 durante la Professional Developers Conference el lenguaje se renombró a C#. Los creadores de Java se quejaron ya que decían que C# era una copia exacta de Java sin ningún tipo de mejora, a lo que Andrés nego y dijo que era mucho más cercano a C++ en su diseño.
En noviembre de 2005 se lanzo C# 2.0 y con el tiempo estos dos lenguajes se han ido dispersando en cuanto a similitud, una de las primeras diferencias fue que C# vino con la adición de genéricos muy diferentes a Java.
Además, C# ha añadido varias características importantes para dar lugar a programación de estilo funcional, extensiones LINQ con C# 3.0 y el empleo de tipos anónimos, métodos de extensión y expresiones lambda. Las extensiones LINQ ayudan a los desarrolladores a reducir mucha cantidad de código repetitivo que está incluido en las tareas comunes como consultar una base de datos, análisis de un archivo xml o búsqueda a través de una estructura de datos, cambiando el énfasis sobre la lógica de programación real para ayudar a mejorar la legibilidad y mantenibilidad.

¿De dónde viene el nombre de C#?

El nombre C# fue inspirado por la notación musical donde una almohadilla(#) indica que la nota escrita debe hacerse un semitono más alto en el tono. Esto es similar al nombre del lenguaje C++ donde (++) indica que una variabre debe ser incrementada en 1. El símbolo de la almohadilla (#) también se asemeja a una ligadura de los cuatros símbolos(+)(en una cuadrícula de dos en dos), implicando que el lenguaje sea un incremento de C++ tal y como se puede observar en esta imagen.

Bibliotecas y Frameworks

Entre las bibliotecas y frameworks estos son los más destacados en C#

  • ASP.NET: es un framework que se utiliza en el desarrollo de aplicaciones web en C#. Proporciona un conjunto completo de herramientas y bibliotecas para la creación de webs modernas, escalables y seguras. Este se divide en dos modelos principales: ASP.NET Web Forms, basado en controles de servidor y un enfoque más parecido a formularios y ASP:NET MVC (Model-Viex-Controller) que se utiliza para la creación de aplicaciones más estructuradas y mantenibles. Además con ASP.NET Core (Versión multiplataforma) se puede desarrollar aplicaciones web que sean compatibles con Windows, Linux y MacOS.
  • Xamarin: es un framework que se utiliza para crear aplicaciones con la capacidad de moverse para Android, iOS y otras plataformas. Este ofrece un desarrollo multiplataforma donde se comparte el código base entre plataformas. Xamarin.Forms permite la creación de interfaces de usuario compartidas que se renderizan de forma independiente en cada plataforma, en cambio Xamarin.Native permite desarrollos más específicos de cada plataforma.
  • Entity Framework:es un ORM (Object-Relational Mapping) que tiene como función la simplificación y la manipulación de bases de datos desde aplicaciones en C#. Permite a los desarrolladores interactuar con los datos utilizando objetos consultas en el lenguaje LINQ. Etity Framework automatiza tareas relacionadas con la base de datos como por ejemplo la generación de consultas SQL, el seguimiento de cambios y la creación de esquemas.
